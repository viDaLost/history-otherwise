Исправление ошибки Telegram: Gemini вернул ответ не в формате JSON

Что исправлено:
1. Backend больше не падает, если Gemini вернул markdown или обычный текст.
2. Добавлен мягкий парсер JSON.
3. Если Gemini вернул текст вместо JSON, backend пробует преобразовать его в JSON вторым запросом.
4. Если второй запрос тоже не дал JSON, backend создаёт безопасный структурированный результат сам.
5. Пользователь больше не увидит ошибку "Gemini вернул ответ не в формате JSON".

Что заменить:
1. api/generate.js
2. package.json

После замены:
1. Загрузи файлы в GitHub.
2. Сделай Redeploy в Vercel.
3. В Telegram открой WebApp заново.
4. Лучше закрыть и открыть чат с ботом, чтобы Telegram не держал старую версию в кэше.
